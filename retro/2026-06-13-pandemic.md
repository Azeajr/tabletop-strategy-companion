## Axis C — Model fit

The `Phase → Category → Condition → Strategy` decomposition is a **strong fit** for Pandemic, which is a linear-arc cooperative game with clear phase progression.

**Phases**: Setup → Early Game → Mid-Game → End-Game follows how Pandemic actually flows. No empty phases; no forced partitions. Coverage is balanced across all four.

**Categories**: Role Synergy, Research Stations, Infection Priority, Cure Priority, Card Sharing, Outbreak Management, Crisis Response, Epidemic Management. Sensible buckets. Some reuse across phases (e.g., "Cure Priority" in Early and Mid, "Crisis Response" in Early, Mid, and End) — this is fine; they're phase-local. Alphabetical sort within each phase works cleanly; no scrambling observed.

**Conditions**: Within each category, conditions sort alphabetically. No reading-order breakage observed (unlike Battleship, which had placement steps scatter). All conditions are distinct and board-matchable in <2s.

**Context filters**: The single filter ("Outbreak track at 4 or above?") encodes a meaningful binary state of Pandemic's game:
- outbreak_track < 4 → "stable" → the team has room to maneuver.
- outbreak_track ≥ 4 → "crisis" → one more chain and the game may be lost.

This is a valid and useful filter for a cooperative game. The **fixed assignments** (A1, A2) now align correctly: crisis-specific strats like "Halt curing — clear cubes" (A7) and "One outbreak = game over" (A14) are marked crisis, and universal/phase-based strats are now null.

**Tags**: 8 of 9 TagEnum values (Offense, Defense, Economy, Pivot, Memory, Bluff, Transition, Closing) are authored but rendered nowhere (Issue #5 scope). Pandemic uses Offense/Defense gently (e.g., role-synergy and placement are defensive; curing is offensive), but the tags are not a gap for this game — the phase/category structure handles the decomposition well.
