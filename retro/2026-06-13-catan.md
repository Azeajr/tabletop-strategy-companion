## Axis C — Model fit

Catan is a linear-arc game (Setup → Early Game → Mid-Game → End-Game). The model fits the game's actual decision flow perfectly.

| # | model element | Finding | Disposition |
|---|---------------|---------|-------------|
| C1 | Phases | Game's natural arc matches declared phases exactly. No empty phases. No forced progression. Setup (settle + secure port) → Early (expand + trade) → Mid (leverage bonuses + scale economy) → End (race to 10 VP). | Accept |
| C2 | Context filter | Single filter (`do_you_have_most_vp`) with "leading" / "trailing" contexts is well-motivated. 2 trailing-specific strategies, 1 leading-specific strategy. No wasted context bits. | Accept |
| C3 | Categories (alphabetical order) | Setup: Initial Placement, Port Strategy. Early: Expansion, Trading. Mid: City Upgrades, Development Cards, Robber, Victory Points. End: Closing, Victory Race. All meaningful, MECE, and alphabetical order matches natural decision sequence. | Accept |
| C4 | Conditions (alphabetical within category) | Within each category, conditions sort alphabetically, which happens to match the player's natural reading order: evaluate → place → expand → trade → bonuses → endgame. No re-ordering needed. | Accept |
| C5 | Stealth row budget (≤8 rows per phase) | Setup: 2 categories + 2 TLDRs = 4 rows. Early: 2 categories + 1 TLDR = 3 rows. Mid: 4 categories + 2 TLDRs = 6 rows. End: 2 categories + 1 TLDR = 3 rows. All ≤8. One-screen fit confirmed. | Accept |
| C6 | Tags | 8 non-TLDR tags authored (Offense, Defense, Economy, Pivot, Memory, Bluff, Transition, Closing) but **rendered nowhere** (grep views confirms). Game's Offense/Defense split is already captured in prose (leading vs trailing context, defensive robber vs offensive expansion). Unrendered tags do not hinder comprehension. | Accept — noted for cross-cutting backlog |
