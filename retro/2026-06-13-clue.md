## Axis C — Model fit

Clue is fundamentally a *deduction game* where progress is driven by information elimination, not temporal phases. The arc-based Phase → Category → Condition → Strategy model is designed for games with a linear progression (setup → early → mid → end). Clue's actual decision structure is: *"What is my current deduction state?" and "What information do I need / can I extract?"* The model fits reasonably, but it's a generalized fit.

**Phases**: Setup, Early, Mid, End are a reasonable learning arc. ✅ Fits.

**Categories**: Somewhat arbitrary. "Suggestion Order", "Information Hiding", "Inference", "Card Tracking", "Narrowing Suspects", "Blocking", "Accusation Timing" don't naturally partition Clue's strategy space — they're a convenient organizational bucketing. A deduction game would more naturally break down as *"My turn: gather info / block opponent / accuse"* and *"Opponent's turn: read and hide info"*. Acceptable, but not MECE.

**Conditions**: Precise and board-matchable, but organized by game phase rather than decision structure. Acceptable.

**Context filters**: The "narrowed" / "wide-open" filter (has the player eliminated 2+ suspects?) makes sense for splitting early broad-spectrum strategies from mid-game focused deduction. ✅ Reasonable fit.

**Tags**: Clue advice is sharply split between Offense (your deductions and accusations) and Defense (your placement and card hiding). The 8 non-TLDR tags are not rendered, so this split goes unsurfaced. If tags were rendered as glyphs (#5), Clue would immediately benefit (e.g., a Bluff/Offense badge on "Showing a card to another player"). Backlog item, not clue-specific.

**Verdict**: The model works for Clue, but the game's decision structure (deduction state + information flow) is a slightly forced fit into the linear arc. The model is a teaching tool (Setup → Early → Mid → End progression), not a reflection of how the game actually flows (non-linear, state-driven). Acceptable, with caveats.

## Notes

- Clue's seed quality is high. The problem is not content, but model fit: a deduction game's decision structure (state → info → deduction) doesn't map perfectly to a linear temporal arc. The seed is a good fit for the current model, and the model is a reasonable generalization that works across 20 games; deeper personalization per game would break the cross-cutting simplicity.
- Next reviewer: if #4 lands (state-based games get a mode/filter option), consider re-modeling Clue's filter to encode "my certainty level" (wide-open → narrowed → ready-to-accuse) and re-score Axis C. Until then, accept as-is.
